#### 测试 4: 与 process.chdir() 方式对比

  files2.push(file);

    - tools\base.ts

  Method 2: process.chdir() + glob.scan()
    - tools\base.ts

## 完整使用示例

### 推荐用法 (使用 cwd 参数)

```typescript
import { Glob } from "bun";
import * as path from "path";

class SearchTool {
  private async globSearch(
    pattern: string,
    searchPath: string,
    maxResults: number,
    followSymlinks: boolean
  ): Promise<ToolResult> {
    const glob = new Glob(pattern);
    const results: string[] = [];
    let count = 0;

    // 直接使用 cwd 参数，无需 process.chdir()
    for await (const file of glob.scan({ 
      cwd: searchPath,      // ← 新方式：传入 searchPath
      followSymlinks, 
      onlyFiles: true 
    })) {
      if (count >= maxResults) break;
      results.push(file);
      count++;
    }

    return this.success(message, {
      pattern,
      path: searchPath,
      files: results,
      count,
      truncated: count >= maxResults,
    });
  }
}
```

### 关键改进点

#### 旧实现 (search.ts 当前代码)
```typescript
const cwd = process.cwd();
try {
  process.chdir(searchPath);  // ← 危险：改变全局状态
  for await (const file of glob.scan({ followSymlinks, onlyFiles: true })) {
    // ...
  }
} finally {
  process.chdir(cwd);         // ← 需要手动恢复
}
```

**问题**:
- 修改全局进程状态 (process.cwd)
- 在并发操作中可能产生竞态条件
- 如果发生异常，可能无法完全恢复

#### 新实现 (使用 cwd 参数)
```typescript
for await (const file of glob.scan({ 
  cwd: searchPath,            // ← 安全：参数化方式
  followSymlinks, 
  onlyFiles: true 
})) {
  // ...
}
```

**优势**:
- ✓ 不修改全局状态
- ✓ 线程安全 / 并发安全
- ✓ 代码更简洁
- ✓ 异常处理更简单
- ✓ 性能无差异 (两种方式完全等价)
